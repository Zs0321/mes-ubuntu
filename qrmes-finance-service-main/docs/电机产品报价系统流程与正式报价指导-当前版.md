# 电机产品报价系统流程与正式报价指导（当前版）

## 三、当前电机产品正式报价流程

### 3.1 入口层

当前主要入口有 4 类：

1. Excel 上传报价
   - 接口：`/api/quote/excel`
   - 入口代码：`app_web/finance_demo.py`
   - 服务层：`ExcelQuoteService.quote_workbook()`

2. 粘贴表格报价
   - 接口：`/api/quote/excel-paste`
   - 入口代码：`app_web/finance_demo.py`
   - 会先将粘贴内容转成临时 workbook，再进入统一报价流程

3. 单物料报价
   - 接口：`/api/quote/single-bom`
   - 内部直接调用：`FinanceSkillQuoteService.quote_items([item], model=...)`

4. AI 报价任务复核
   - 接口：`/api/quote/ai-route`
   - 最终仍围绕统一 payload 汇总分析结果

### 3.2 主链路步骤

#### 步骤 1：输入标准化与模式归一化

在 `quote_items()` 中先做：

- `production_mode` 归一化：仅保留 `sample` 与 `mass`
- `annual_volume` 归一化：量产口径明确为 `年产量`
- `_prepare_items(items)`：标准化 BOM 行

#### 步骤 2：训练参考回填

在 `_prepare_items()` 中优先命中训练数据：

- 文件：`app_web/backend/data/liugong_motor_training_reference.json`
- 优先级：
  1. 物料编码
  2. 名称 + 规格
  3. 名称唯一时按名称命中

可回填字段：

- 材质
- 重量
- 工艺
- 当前参考单价

#### 步骤 3：金蝶采购参考回填

主链路会回填：

- 金蝶参考价
- 税价
- 供应商
- 最近采购日期

该部分主要用于：

- 财务参考路线
- 差异分析
- 供应商信息展示

#### 步骤 4：AI staged 输入增强

`_prepare_ai_staged_skill_items()` 负责：

- 名称规格价格带识别
- 名称规格估重
- AI 高置信工艺 / 材质 / 重量覆盖
- 形成 `skills 实际采用输入`

关键输出包括：

- `skills_input_process`
- `skills_input_material`
- `skills_input_weight_kg`
- 及对应来源字段

#### 步骤 5：skills 脚本计价

主链路会生成临时 `input.xlsx` 并执行 `changjiang-bom-pricing` 相关脚本，主要包括：

- `price_bom_xlsx.py`
- `model_volume_pricing.py`（量产模式下）
- gap / 格式化脚本（按计划启用）

输出包括：

- line 级 rules 报价
- grouped 结果
- 量产三档价格
- 在线价格快照

#### 步骤 6：AI 路线补位与复核

系统在 skills 结果基础上继续做：

- AI 建议工艺
- AI 建议材质
- AI 报价补位
- 与财务参考路线做差异分析

#### 步骤 7：量产开模路线覆盖

当 `production_mode == mass` 时，主链路会额外尝试进入量产开模公式。

当前已接入的量产开模主类：

1. 高精轴
2. 铁芯冲压模 / 叠压模
3. 精密端盖
4. 机壳
5. 接线盒 / 盖板 / 挡板 / 压盖 / 轴套等壳体与 generic 金属件

命中后会补充：

- `sample_machining_unit_price`
- `mass_tooling_unit_price`
- `tooling_cost`
- `mass_break_even_volume`
- `mass_process_route`
- `volume_pricing_summary`

## 四、当前电机产品各主要类别口径

### 4.1 定子铁芯 / 转子铁芯

当前已进入：

- `冲压模 + 叠压` 量产口径

注意：

铁芯的小批路线通常不是传统 CNC 机加工，而是更接近：

- 小批冲压
- 叠压
- 固化 / 焊接 / 铆接

因此铁芯的样品/小批路线价与量产价会有差距，但未必像机壳那样差得很大。

### 4.2 机壳

当前已进入：

- 拉伸 / 型材 / 开模后单件摊薄口径

机壳的小批路线更接近：

- 拉伸后 CNC 精加工
- 表处 / 后加工摊销

因此机壳样品价与量产价差通常更明显。

### 4.3 端盖

当前已进入：

- 精密端盖量产开模口径

已纳入的维度包括：

- 铸造毛坯
- 轴承室 / 止口 / 安装面 / 螺纹孔机加工
- 密封槽
- 表处

### 4.4 电机轴

当前已进入：

- 高精轴量产开模口径

主要考虑：

- 材料
- 锻造毛坯
- 车削 / 精加工
- 热处理
- 磨削
- 花键 / 键槽

### 4.5 接线盒 / 挡板 / 压盖 / 轴套 / 盖板

当前已进入：

- 壳体 / generic 金属件量产开模口径

本次已验证：

- 接线盒已支持量产开模
- 转子轴套已支持量产开模
- 挡板 / 压盖 / 盖板类已支持量产开模

## 五、当前已确认的注意事项

### 5.1 `样品机加工单价` 这一列名称并不完全准确

当前系统统一沿用字段名：

- `sample_machining_unit_price`

但对不同物料，其实际语义不同：

- 铁芯：更接近样品/小批 `冲压叠压` 路线价
- 机壳 / 端盖 / 电机轴：更接近样品/小批 `机加工 / 精加工` 路线价

因此审核时更准确的理解应为：

- `样品/小批路线单价`

### 5.2 行级报价总表中的“工艺”与 AI 分析明细中的“AI建议工艺”不是同一字段

- **行级报价总表**中的 `工艺`：来自当前 item 的 `process`，更接近系统最终采用工艺
- **AI分析明细**中的 `AI建议工艺`：来自 `ai_route_process_guess`，表示 AI 推断建议工艺，不一定等于最终采用工艺

因此两者可能相同，也可能不同。

### 5.3 页面若不是直接挂在 finance_demo 报价后端下，可能出现接口基址问题

当前前端任务查询已支持通过以下方式指定接口基址：

- `window.FINANCE_DEMO_API_BASE`
- `document.body.dataset.financeApiBase`

若页面挂在其他服务下，又未显式设置该基址，就可能出现：

- 当前页面未连接到 finance_demo 报价接口

这属于页面请求基址问题，不是报价逻辑本身失效。

## 六、当前检查后对系统合理性的判断

### 6.1 当前主逻辑没有发现需要立即推翻的错误

本次检查后，当前系统更像是：

- 主链路逻辑成立
- 量产开模覆盖已较完整
- 训练参考命中有效
- 导出链路可用

当前更需要继续优化的方向主要是：

- 展示语义清晰度
- 审核可解释性
- 前后端接口基址一致性

### 6.2 价格差异不能机械判断“都应该很大”

公开制造业规律与当前系统实现都表明：

- 模具 / 开模路线前期模具费高
- 量产后单件通常下降
- 但是否“差很多”，取决于：
  1. 样品路线本身是否已接近量产工艺
  2. 当前年产量是否达到平衡点

例如：

- 铁芯：样品路线本身就接近冲压叠压，价差可中等
- 机壳：样品路线更接近机加 / 精加工，价差通常更大

## 七、业务使用建议

### 建议 1：量产模式下必须明确 `annual_volume` 是年产量

当前系统中：

- `annual_volume` = `年产量`
- 不是单次采购量

### 建议 2：审核时优先看这几项

建议审核顺序：

1. 训练参考是否命中
2. 最终采用工艺是否合理
3. 年产量是否填写正确
4. 是否达到开模平衡点
5. AI 路线与财务参考差异是否过大

### 建议 3：不要把所有样品路线都理解成机加工

特别是：

- 定子铁芯
- 转子铁芯
- 某些冲压 / 压铸 / 小批成形件

这些件的小批路线通常已接近量产路线，不能简单按传统 CNC 机加工理解。
