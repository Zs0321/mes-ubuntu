## 目标

建立从采购、收货、来料检、入库、配料、装配、PCBA 转换、测试到出货的全链路追溯体系，做到来源可查、去向可追、过程可还原、责任可定位。

## 本期范围

- 采购单：作为外部来源起点。
- 来料物料批次：形成内部追溯主键。
- 包装单元：盘、箱、袋、盒、单件。
- 来料检记录：绑定到批次。
- 仓位库存：区分待检区与正式仓位。
- 生产产品 SN：成品追溯主线。
- PCBA 批次/单板码：中间转换对象。
- 出货单/箱码：客户交付追溯出口。

优先跑通四个核心页面：收货贴标、来料检绑定、配料扫码、产品装配绑定。

## 主链路

```text
采购单 -> 收货单 -> 物料批次码 -> 包装码 -> 来料检记录 -> 库存记录
库存记录 -> 配料记录 -> 产品SN / PCBA批次 -> 测试记录 -> 箱码 -> 出货单
```

## 编码规则

### 批次码

```text
ML|{物料编码}|{供应商编码}|{到货日期YYYYMMDD}|{4位流水号}
示例：ML|MAT001|SUP008|20260415|0003
```

### 包装码

```text
PK|{批次码}|{2位包装序号}
示例：PK|ML|MAT001|SUP008|20260415|0003|02
```

### 单件序列号

```text
SN|{物料编码}|{日期YYYYMMDD}|{6位流水号}
```

### PCBA 批次码

```text
PCBA|{产线编码}|{日期YYYYMMDD}|{4位流水号}
```

## 标签打印规则

- 电子料：盘 / 袋 / 盒，以批次追溯为主。
- 结构件：箱 / 包，关键件可扩展到单件。
- 线束件：单件优先，其次包装。
- 标准件：整箱 / 整包，一般无需单件码。
- PCBA：单板码或批次码，作为新物料重新入库。

## 二维码 payload 建议字段

至少包含：

- `code_type`: BATCH / PACK / SN / PRODUCT / PCBA
- `code`: 当前二维码对应编码
- `material_code`
- `supplier_code`
- `batch_code`
- `pack_index`
- `qty`
- `unit`
- `trace_mode`

## 第一阶段页面和动作

### 收货贴标

输入采购单、供应商二维码、到货数量、包装数量；系统生成内部批次码、包装码、收货记录、待检库存记录、二维码打印任务。

### 来料检绑定

扫描批次码或包装码，带出采购单、供应商、物料信息、到货数量、收货日期和检验模板；提交合格、让步接收或不合格，绑定检验报告并更新批次状态。

### 入库

扫描批次码或包装码、正式仓位码，确认入库数量；库存状态从待检转可用，仓位切换，生成入库记录。

### 配料扫码

扫描产品码/工单码和物料包装码/批次码；校验 BOM、状态，生成配料记录，建立产品码与物料码预绑定，扣减或冻结库存。

### 产品装配绑定

扫描产品码和物料码，必要时录入数量、序列号、安装位置；建立产品 SN 到物料批次/单件序列号关系。

## 后续阶段

- PCBA 投料与产出：建立电子料批次 -> PCBA 批次转换关系，PCBA 作为新库存物料入库。
- 测试绑定：产品 SN -> 测试结果/报告。
- 打包出货：出货单 -> 箱码 -> 产品 SN。
- 正反向追溯查询与质量异常批量反查。

## 异常场景

- 采购单不存在：禁止收货并提示重新同步。
- 来料数量大于采购剩余数量：预警或禁止，按权限放行。
- 供应商二维码无法解析：允许保存原始码并手动确认物料。
- 无检验模板：阻止来料检提交并提示维护模板。
- 不合格批次入库/出库：强制拦截。
- 物料不属于 BOM：报警并禁止绑定。
- 冻结批次继续使用：拦截并记录异常。
- 包装码重复打印：记录重打原因和操作人。
- 线边余料回仓：生成退料和回仓事件。

## 验收标准

1. 任意产品 SN 可追溯到关键物料批次、来料检记录、采购单和供应商。
2. 任意物料批次可反查使用到哪些产品、工单、PCBA 或出货单。
3. 扫码收货到贴标过程无需手工创建主编码。
4. 来料检、入库、配料、装配四个主流程可全程留痕。
5. 所有关键动作均可查询操作人、时间、设备和结果。
