# 物料追溯执行：金蝶 BOS 新增字段最终建议

## 1. 当前口径

本次物料追溯执行按以下口径设计：

1. 不需要单件序列号。
2. 不新增组件序列号字段。
3. 批次码共用金蝶已有批次码。
4. 工单号共用金蝶生产订单/工单号。
5. 采购单号、物料、供应商、数量、单位、仓库、人员等，优先共用金蝶已有字段。
6. MES 本地继续保存完整追溯明细，金蝶 BOS 只补充金蝶标准字段无法表达的追溯字段。

因此，本次不建议新增：

- `F_MES_BatchCode`
- `F_MES_ComponentSn`
- `F_MES_AssemblyBindId`
- `F_MES_PrintStatus`
- `F_MES_PackageIndex`
- `F_MES_SourcePayload`

当前最小新增字段集合为：

1. `F_MES_PackageCode`
2. `F_MES_TraceId`
3. `F_MES_SupplierQrRaw`
4. `F_MES_IqcAttachmentIds`

可选字段：

5. `F_MES_QrCode`

---

## 2. 不需要新建金蝶数据库物理表

当前不建议直接在金蝶数据库里手工新建物理表。

原因：

1. 金蝶业务对象由 BOS 元数据管理，手工建表不等于金蝶业务对象。
2. 手工表无法自然进入金蝶权限、单据、WebAPI、审计和升级体系。
3. 后续金蝶升级、补丁、二开维护会有风险。

正确做法：

1. 通过 BOS 在现有金蝶对象上增加自定义字段。
2. MES 本地 SQLite 保存完整追溯明细。
3. 金蝶只保存关键追溯字段和关联 ID。
4. 后续如果确实要求金蝶内完整查询追溯链，再通过 BOS 新建自定义业务对象，不直接 SQL 建表。

## 3. 最小 BOS 新增字段清单

### 3.1 F_MES_PackageCode

| 项目 | 内容 |
|---|---|
| 字段编码 | `F_MES_PackageCode` |
| 字段名称 | MES包装码 |
| 字段类型建议 | 文本 |
| 长度建议 | 100 或 200 |
| 是否必加 | 必加 |
| 对应 MES 字段 | `package_code` |
| 典型内容 | PK 包装码 |

用途：

保存 MES 生成的 PK 包装码。

说明：

金蝶已有批次码可以用于批次追溯，但不一定能表达“每箱、每包、每个包装单元”的追溯码。所以包装码需要单独新增。

建议加在：

1. 采购收货/收料对象。
2. 采购入库对象。
3. 生产领料/用料对象。
4. 如后续使用批序关系或追溯关系对象，也建议加。

---

### 3.2 F_MES_TraceId

| 项目 | 内容 |
|---|---|
| 字段编码 | `F_MES_TraceId` |
| 字段名称 | MES追溯ID |
| 字段类型建议 | 文本 |
| 长度建议 | 100 |
| 是否必加 | 建议必加 |
| 对应 MES 字段 | `trace_event_log.id` 或 MES 追溯主记录 ID |

用途：

让金蝶单据能够反查 MES 本地追溯记录。

说明：

完整追溯链不建议全部塞进金蝶字段。金蝶单据只保存 `F_MES_TraceId`，需要查看完整链路时再回 MES 追溯库查询。

建议加在：

1. 采购收货/收料对象。
2. 采购入库对象。
3. IQC/检验对象。
4. 生产领料/用料对象。

---

### 3.3 F_MES_SupplierQrRaw

| 项目 | 内容 |
|---|---|
| 字段编码 | `F_MES_SupplierQrRaw` |
| 字段名称 | 供应商原始二维码 |
| 字段类型建议 | 长文本 |
| 是否必加 | 建议加 |
| 对应 MES 字段 | `supplier_qr_raw` |

用途：

保存供应商来料二维码的原始内容。

说明：

供应商二维码格式不统一，金蝶标准字段通常没有固定承载位置。该字段用于后续追溯供应商原码与 MES 包装码、金蝶批次码之间的关系。

建议加在：

1. 采购收货/收料对象。
2. IQC/检验对象，若检验时需要查看供应商原始二维码。

---

### 3.4 F_MES_IqcAttachmentIds

| 项目 | 内容 |
|---|---|
| 字段编码 | `F_MES_IqcAttachmentIds` |
| 字段名称 | MES检验附件ID |
| 字段类型建议 | 长文本 |
| 是否必加 | 建议加 |
| 对应 MES 字段 | `iqc_attachment.attachment_id`、`iqc_record.attachments_json` |

用途：

保存 IQC 检验附件 ID、附件链接或附件清单。

说明：

不要把附件 base64 内容写入金蝶普通字段。金蝶侧只保存附件引用；附件内容继续保存在 MES 本地库，或后续对接金蝶附件机制。

建议加在：

1. IQC/检验对象。

---

## 4. 可选 BOS 新增字段

### 4.1 F_MES_QrCode

| 项目 | 内容 |
|---|---|
| 字段编码 | `F_MES_QrCode` |
| 字段名称 | MES二维码内容 / 追溯链接 |
| 字段类型建议 | 长文本 |
| 是否必加 | 可选 |
| 对应 MES 字段 | `qr_payload_json` 或追溯 URL |

用途：

保存二维码内容、追溯链接或简化后的二维码字符串。

建议：

如果金蝶只需要能反查追溯链，则不一定需要该字段，使用 `F_MES_TraceId` + `F_MES_PackageCode` 即可。

如果希望金蝶单据上直接显示二维码内容、扫码链接或追溯 URL，则可以增加该字段。

注意：

不建议把完整的大 JSON payload 全部写入金蝶字段。更推荐保存短二维码字符串或追溯链接。

建议加在：

1. 采购收货/收料对象。
2. 采购入库对象。
3. 标签相关对象，若现场有对应金蝶对象。

---

## 5. 当前明确不新增的字段

### 5.1 F_MES_BatchCode

| 项目 | 内容 |
|---|---|
| 字段编码 | `F_MES_BatchCode` |
| 字段名称 | MES批次码 |
| 当前建议 | 不新增 |

原因：

当前口径是“共用批次码”。也就是 MES 批次追溯直接使用金蝶已有批次码，不再单独新增 MES 批次码字段。

只有在未来确认“ML 批次码不是金蝶正式批次码，只是 MES 内部追溯码”时，才需要重新考虑新增该字段。

---

### 5.2 F_MES_ComponentSn

| 项目 | 内容 |
|---|---|
| 字段编码 | `F_MES_ComponentSn` |
| 字段名称 | MES组件序列号 / 单件序列号 |
| 当前建议 | 不新增 |

原因：

当前明确不需要单件序列号，追溯共用批次码即可。

---

### 5.3 F_MES_AssemblyBindId

| 项目 | 内容 |
|---|---|
| 字段编码 | `F_MES_AssemblyBindId` |
| 字段名称 | MES装配绑定ID |
| 当前建议 | 第一阶段不新增 |

原因：

当前不做单件序列号装配绑定。产品追溯先按工单、产品、物料、批次码、包装码进行追溯，装配明细保留在 MES 本地即可。

如果后续要求金蝶内完整查询装配追溯链，再考虑新增自定义对象或关联字段。

---

### 5.4 F_MES_PrintStatus

| 项目 | 内容 |
|---|---|
| 字段编码 | `F_MES_PrintStatus` |
| 字段名称 | 标签打印状态 |
| 当前建议 | 不新增 |

原因：

打印状态属于 MES 执行层。当前二维码打印尚未直连斑马打印机，打印任务状态保存在 MES 本地 `label_print_task` 和 `material_qrcode` 更合适。

---

### 5.5 F_MES_PackageIndex

| 项目 | 内容 |
|---|---|
| 字段编码 | `F_MES_PackageIndex` |
| 字段名称 | MES包装序号 |
| 当前建议 | 不新增 |

原因：

包装序号可以从 PK 包装码规则里解析，第一阶段没必要单独新增字段。

---

### 5.6 F_MES_SourcePayload

| 项目 | 内容 |
|---|---|
| 字段编码 | `F_MES_SourcePayload` |
| 字段名称 | MES原始数据 |
| 当前建议 | 不新增 |

原因：

完整 JSON payload 留在 MES 本地追溯库，不建议塞入金蝶普通字段。

## 6. 按金蝶对象放置建议

### 6.1 采购收货 / 收料对象

待确认真实 FormId：

- `PUR_ReceiveBill` 或现场实际采购收料对象。
- 如果现场不使用收料单，也可能直接在采购入库对象上处理。

建议新增字段：

| 字段编码 | 字段名称 | 是否必须 |
|---|---|---|
| `F_MES_PackageCode` | MES包装码 | 必加 |
| `F_MES_TraceId` | MES追溯ID | 建议加 |
| `F_MES_SupplierQrRaw` | 供应商原始二维码 | 建议加 |
| `F_MES_QrCode` | MES二维码内容 / 追溯链接 | 可选 |

不新增，直接复用金蝶字段：

| 数据 | 复用对象/字段 |
|---|---|
| 采购单号 | 采购订单号 |
| 物料编码 | `BD_MATERIAL` |
| 供应商 | `BD_Supplier` |
| 数量 | 收货数量字段 |
| 单位 | `BD_UNIT` |
| 批次码 | 金蝶已有批次码 |
| 操作员 | `BD_Empinfo` 或金蝶制单/审核人字段 |

---

### 6.2 采购入库对象

可能对象：

- `STK_InStock`
- 或现场实际采购入库单对象

建议新增字段：

| 字段编码 | 字段名称 | 是否必须 |
|---|---|---|
| `F_MES_PackageCode` | MES包装码 | 必加 |
| `F_MES_TraceId` | MES追溯ID | 建议加 |
| `F_MES_QrCode` | MES二维码内容 / 追溯链接 | 可选 |

不新增，直接复用金蝶字段：

| 数据 | 复用对象/字段 |
|---|---|
| 物料编码 | `BD_MATERIAL` |
| 数量 | 入库数量字段 |
| 单位 | `BD_UNIT` |
| 仓库 | `BD_STOCK` |
| 库存状态 | `BD_STOCKSTATUS`，按现场配置确认 |
| 批次码 | 金蝶已有批次码 |

---

### 6.3 IQC / QMS 检验对象

真实 FormId 仍需从现场金蝶确认。

建议新增字段：

| 字段编码 | 字段名称 | 是否必须 |
|---|---|---|
| `F_MES_TraceId` | MES追溯ID | 建议加 |
| `F_MES_IqcAttachmentIds` | MES检验附件ID | 建议加 |
| `F_MES_SupplierQrRaw` | 供应商原始二维码 | 视检验页面是否需要查看 |
| `F_MES_PackageCode` | MES包装码 | 视是否按包装码检验 |

不新增，直接复用金蝶字段：

| 数据 | 复用对象/字段 |
|---|---|
| 检验员 | `BD_Empinfo` |
| 检验结果 | 金蝶质量结果枚举，需现场确认 |
| 检验报告号 | 如果金蝶已有报告号字段，则复用 |
| 物料编码 | `BD_MATERIAL` |
| 供应商 | `BD_Supplier` |
| 批次码 | 金蝶已有批次码 |

特别注意：

- `content_base64` 不写金蝶。
- 附件内容不写普通字段。
- 金蝶侧只保存附件 ID、附件链接或附件清单。

---

### 6.4 生产领料 / 用料对象

可能相关对象：

- `PRD_MO` 生产订单。
- `PRD_PPBOM` 生产用料清单。
- 生产领料单真实 FormId 待确认。

建议新增字段：

| 字段编码 | 字段名称 | 是否必须 |
|---|---|---|
| `F_MES_PackageCode` | MES包装码 | 必加 |
| `F_MES_TraceId` | MES追溯ID | 建议加 |

不新增，直接复用金蝶字段：

| 数据 | 复用对象/字段 |
|---|---|
| 工单号 | `PRD_MO` 生产订单号/工单号 |
| 物料编码 | `BD_MATERIAL` |
| 数量 | 领料数量字段 |
| 单位 | `BD_UNIT` |
| 批次码 | 金蝶已有批次码 |
| 领料人/操作员 | `BD_Empinfo` |

说明：

金蝶已有工单号，所以 `work_order_no` 不新增字段，直接引用 `PRD_MO`。

---

### 6.5 产品追溯 / 装配追溯

当前口径：

1. 不做单件序列号。
2. 共用批次码。
3. 追溯到产品时，用产品、工单、物料、批次码、包装码建立链路。
4. 完整绑定关系先保留在 MES 本地。

第一阶段建议：

不新建金蝶装配追溯对象。

如果后续确实要求金蝶内也完整保存追溯链，再考虑 BOS 自定义对象。

可选自定义对象：

`MES_TraceBind`：MES装配追溯绑定关系。

但按当前需求，第一阶段暂不需要。

## 8. 一句话结论

当前金蝶 BOS 第一阶段只需要新增 4 个字段：

1. `F_MES_PackageCode`
2. `F_MES_TraceId`
3. `F_MES_SupplierQrRaw`
4. `F_MES_IqcAttachmentIds`

可选新增：

1. `F_MES_QrCode`

不新增批次码字段，因为批次码共用金蝶已有批次码。

不新增单件序列号字段，因为本次不需要单件序列号。

不新建金蝶数据库物理表；如未来需要金蝶完整保存装配追溯链，再通过 BOS 新建自定义业务对象。
