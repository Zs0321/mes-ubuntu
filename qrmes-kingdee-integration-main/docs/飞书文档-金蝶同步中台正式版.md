## 1. 文档目标

本文档用于说明当前 `qrmes-kingdee-integration` 仓库已经完成的“金蝶同步中台”能力，包括：
1. 已支持的真实联通能力
2. 当前尚未支持或仅完成骨架的能力
3. 本地数据库中台的结构和同步方式
4. MES 框架下如何调用本地数据库 API，而不是直接调用金蝶

该中台的核心原则是：
- 第三方业务系统不直接操作金蝶
- 第三方统一操作本地数据库 API
- 金蝶同步由 `qrmes-kingdee-integration` 仓库负责执行
- 本地数据作为 MES / QRMES 的金蝶缓存与交换层

---

## 2. 当前总体架构

### 2.1 组成

当前中台由三部分组成：

1. 金蝶通用客户端层
- 登录：`LoginBySign`
- 查询：`ExecuteBillQuery`
- 查看：`View`
- 写回：`Save`

2. 本地同步数据库层
- SQLite 数据库文件：`/Volumes/172.16.30.10/volume2/MES/QRMES/kingdee_sync.db`
- 数据对象表：`kingdee_objects`
- 出站变更队列表：`kingdee_change_queue`

3. 本地 DB API 服务层
- 服务地址：`http://127.0.0.1:9010`
- 提供健康检查、拉取同步、本地数据查询、本地修改回写入口

### 2.2 同步方向

当前支持两类同步：

A. 金蝶 -> 本地数据库
- 从金蝶通过 API 拉取数据
- 写入 `kingdee_sync.db`
- MES / 其他服务从本地 API 读取

B. 本地数据库 -> 金蝶
- 第三方调用本地 API 提交修改
- 中台将修改写入本地队列表
- 中台调用金蝶 `Save` 接口回写
- 回写成功后更新本地对象状态

## 5. MES 框架应如何调用本地 DB API

## 5.1 推荐原则

在 MES / QRMES 框架下，后续所有业务调用都应遵循以下原则：

1. 不直接请求金蝶 WebAPI
2. 不在业务仓中重复实现登录、签名、查询、保存逻辑
3. 统一请求本地中台 API：`http://127.0.0.1:9010`
4. 由中台负责：
   - 查询金蝶
   - 缓存到本地数据库
   - 回写金蝶
   - 统一审计和队列管理

---

## 5.2 MES 内查询示例

resp.raise_for_status()

```

### 提交本地生产订单修改，由中台回写金蝶

    'Number': 'MO001063',

}

    'http://127.0.0.1:9010/api/local-db/production_order/MO001063',
    json=payload,
    timeout=60,

print(resp.json())
```

### 方式 B：通过 qrmes-web-core 代理入口调用（暂缓切换）

`qrmes-web-core` 后端通过 HTTP 调用 `qrmes-kingdee-integration` 本地 API，是后续 MES 侧接入中台的目标形态之一，但当前阶段暂不切换 Web 端。

暂缓原因：
- 金蝶同步中台仍有追溯树、授权受限对象、回写闭环等未完全打通项；
- `qrmes-web-core` 是现有 MES 主服务，不应在中台能力未稳定前切换成熟链路；
- 当前文档只保留该接入方式作为后续方案，不作为当前实施动作。

后续只有在以下条件满足后，再评估启用 web-core 代理入口：
1. 中台核心数据集和追溯能力完成验收；
2. 本地 API、自动同步、回写队列在 9010 环境连续稳定运行；
3. 已完成最小回归测试，并明确不会影响 8891 正式服务；
4. 用户明确要求切换 Web 端接入。

## 5.3 在 mes 框架内的接入建议（后续阶段）

后续若进入 MES 接入阶段，可在 `qrmes-v2.0` 或 split 后的 `qrmes-web-core` 中新增一层轻量客户端，例如：

```python
class KingdeeLocalDbApiClient:
    def __init__(self, base_url='http://127.0.0.1:9010'):
        self.base_url = base_url.rstrip('/')

    def list_materials(self):
        return requests.get(f'{self.base_url}/api/local-db/material', timeout=30).json()

    def list_production_orders(self):
        return requests.get(f'{self.base_url}/api/local-db/production_order', timeout=30).json()

    def pull_production_orders(self):
        return requests.post(f'{self.base_url}/api/sync/production-orders/pull', timeout=60).json()

    def update_production_order(self, bill_no, payload):
        return requests.post(
            f'{self.base_url}/api/local-db/production_order/{bill_no}',
            json=payload,
            timeout=60,
        ).json()
```

这样业务仓只面向本地 API，金蝶细节全部收敛到中台仓。

当前 `qrmes-kingdee-integration` 已经不是纯设计阶段，而是已经具备了“金蝶同步中台”的第一版可运行能力：
