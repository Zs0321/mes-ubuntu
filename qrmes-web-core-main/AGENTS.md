# AGENTS 接入片段

这段内容可直接复制到目标项目根目录的 `AGENTS.md` 中，作为团队 AI 协作规范入口。

```md
## Team AI Workflow


- 在实施非琐碎变更之前，请先阅读`docs/ai/README.md`。
- 使用`docs/ai/TEAM_AI_COLLABORATION.md`作为团队级工作流程的基准。
- 在更改产品行为之前，请先阅读项目的`REQUIREMENTS.md`文件。
- 如果`REQUIREMENTS.md`缺失或明显不完整，请暂停直接实现，先帮助起草该文档。
- 在实施之前，明确变更边界：哪些内容可以变更，哪些内容应避免变更，哪些内容未经确认不得变更。
- 对于大型或高风险的任务，将工作分解为可验证的子任务，并在关键检查点暂停以进行测试或确认。
- 当任务涉及高风险模块时，在进行广泛编辑之前，帮助起草或审查针对特定模块的规则。
- 当任务涉及角色、权限、变更边界或需求同步时，请主动遵循`docs/ai/skills/`下的相关工作流程。
- 不要指望工程师能记住模板名称。代理应根据任务类型主动引导工作流程。
```

## 推荐使用方式

1. 把这段内容接入目标项目根 `AGENTS.md`
2. 确保项目里已有 `docs/ai/README.md`
3. 第一次接入时，把 `BOOTSTRAP_PROMPT.md` 里的提示词发给 AI
