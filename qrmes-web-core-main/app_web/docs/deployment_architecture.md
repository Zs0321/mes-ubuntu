### 1.2 分离策略

| 数据类型 | 存储位置 | 原因 |
|---------|---------|------|
| **照片文件** | NAS (`<DATA_DIR>/picture/`) | 大容量、长期保存、多端访问 |
| **配置文件** | NAS (`<DATA_DIR>/*.json`) | 集中管理、备份方便 |
| **记录文件** | NAS (`<DATA_DIR>/record/`) | 历史数据、审计需求 |
| **SQLite 数据库** | 服务器本地 (`/data/databases/`) | 高并发读写、性能优先 |
| **日志文件** | 服务器本地 (`/data/logs/`) | 频繁写入、快速访问 |
| **缓存数据** | 服务器本地 + Redis | 性能优化 |

### 1.3 关键优势

✅ **存储与计算分离**  
   - Web 服务器专注业务逻辑处理  
   - NAS 专注大容量文件存储和备份

✅ **灵活扩展**  
   - 服务器可随时升级配置  
   - NAS 可按需扩容存储空间  
   - 支持多台 Web 服务器共享同一 NAS

✅ **数据安全**  
   - NAS 提供 RAID 保护和快照备份  
   - 服务器数据可独立备份  
   - 网络访问控制和身份验证

✅ **性能优化**  
   - 高频数据库操作走本地 SSD  
   - 大文件读取走 NAS（千兆/万兆网络）  
   - Redis 缓存热点照片路径

### 3.2 端口规划

| 服务 | 内部端口 | 外部端口 | 说明 |
|------|---------|---------|------|

## 5. 配置变更

### 5.1 config.py 修改建议

```python
# config.py 新增配置项

DEFAULT_CONFIG = {
    # ... 保持现有配置 ...
    
    # 新增：部署模式配置
    "deployment_mode": "production",  # "development" 或 "production"
    
    # 新增：NAS 挂载点（在服务器上的本地路径）
    "nas_mount_point": "/opt/qrmes/nas-mount/qrmes",
    
    # 修改：照片基础路径（指向服务器上的 NAS 挂载点）
    "photo_base_path": "/opt/qrmes/nas-mount/qrmes/picture",
    
    # 修改：数据库路径（指向服务器本地）
    "database_path": "/opt/qrmes/data/databases",
    
    # 修改：日志路径
    "log_path": "/opt/qrmes/data/logs",
    
    # 新增：Redis 配置
    "redis_host": "redis",  # Docker Compose 服务名
    "redis_port": 6379,
    "redis_db": 0,
    "redis_password": None,
}
```

### 5.2 环境变量配置

通过 `.env` 文件管理敏感配置：

```bash
# NAS 连接配置
NAS_SERVER=172.16.30.2
NAS_SHARE_NAME=mes
NAS_USERNAME=your_username
NAS_PASSWORD=your_password
NAS_BASE_PATH=QRMES

# 数据库配置
DATABASE_PATH=/opt/qrmes/data/databases

# 日志配置
LOG_PATH=/opt/qrmes/data/logs
LOG_LEVEL=INFO

# Redis 配置
REDIS_HOST=redis
REDIS_PORT=6379
REDIS_DB=0

# 群晖 API 配置
SYNOLOGY_API_URL=https://172.16.30.2:5001
SYNOLOGY_API_VERIFY_SSL=true

# Flask 配置
FLASK_ENV=production
FLASK_SECRET_KEY=your_secret_key_here
FLASK_DEBUG=False
```

## 6. 部署优势

### 6.1 vs 当前架构

| 对比项 | 当前架构 | 新架构 | 改进 |
|-------|---------|--------|------|
| 部署位置 | 群晖 NAS | 独立服务器 | ✅ 性能提升 2-5x |
| 照片访问 | 本地读取 | 网络挂载 | ✅ 灵活性提升 |

| 会话管理 | Flask 默认 | Redis | ✅ 支持多实例 |
| 扩展性 | 单点 | 可水平扩展 | ✅ 支持负载均衡 |
| 备份策略 | 统一备份 | 分层备份 | ✅ 更灵活 |

## 8. 风险评估

| 风险 | 影响 | 概率 | 缓解措施 |
|------|------|------|----------|
| NAS 挂载失败 | 无法访问照片 | 中 | 提供 WebDAV 降级方案 |
| 网络延迟 | 照片加载慢 | 低 | Redis 缓存 + 预加载 |
| 数据库迁移丢失 | 数据丢失 | 低 | 迁移前完整备份 |
| Docker 学习成本 | 部署困难 | 中 | 提供详细文档和脚本 |
| 服务器硬件故障 | 服务中断 | 低 | 定期备份 + 快速恢复脚本 |
