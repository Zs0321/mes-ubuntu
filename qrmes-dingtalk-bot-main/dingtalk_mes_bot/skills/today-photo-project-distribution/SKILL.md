# 今日工序照片项目分布

## 概述
这个 skill 描述了钉钉 MES 机器人如何回答“今天上传的工序照片按项目分布”的问题。

适用的典型问法包括：
- 今天工序照片分布在几个项目
- 每个项目多少张照片
- 这 1255 张工序照片分别属于哪些项目

这个 skill 同时定义了本地大模型兜底回复的安全规则：
- 不能把 `Drafting` 这类草稿内容发回群里
- 不能把 `Internal Monologue` 这类内部独白发回群里
- 不能把 `Thinking Process` 这类思考过程发回群里
- 一旦命中这些内容，直接丢弃模型回复，回退到机器人安全兜底

## 何时使用
- 用户问的是“今天/今日”上传的工序照片统计问题。
- 用户想知道这些照片分布在几个项目、每个项目多少张。
- 机器人准备调用本地文本模型兜底，但必须先对输出做严格清洗。

不要把这个 skill 用在图片标签 OCR 识别场景，那属于二维码标签识别 skill。

## 真实查询规则
1. 从 `/volume2/MES/QRMES/unified.db` 读取当天照片记录。
2. 使用表 `process_photos`。
3. 用 `captured_at` 过滤到当天 00:00:00 到次日 00:00:00 之间的数据。
4. 逐条解析 `metadata` JSON。
5. 优先使用 `projectName` 作为项目分组键。
6. 如果 `projectName` 为空，则回退到 `projectCode`。
7. 如果两者都为空，则归到 `未标注项目`。
8. 按照片数量倒序、项目名正序排序。
9. 最终用中文返回：
   - 今天涉及的项目数
   - 今天工序照片总数
   - 每个项目的照片数量

## 回复格式
统一按下面的结构回复：

```text
今天工序照片共分布在 X 个项目，共 Y 张。
1. 项目A：N 张
2. 项目B：M 张
```

## 大模型安全规则
当机器人走本地模型兜底时：
- 如果返回内容里包含 `Drafting`
- 或 `Internal Monologue`
- 或 `Thinking Process`
- 或类似的草稿、分析、推理标记

则这条模型回复必须直接丢弃。

不要尝试把这些草稿翻译给群里。
不要把思维链内容展示给用户。

## 当前限制
这个 skill 当前只负责“在钉钉群里回答统计结果”。
它不会自动创建或更新钉钉文档。
如果用户说“统计到文档中”，当前版本仍然只会先返回统计结果，除非后续再补文档写入动作流。

## 快速验证
1. 提问：`今天上传了多少张工序照片`
2. 提问：`这1255张工序照片分布在几个项目？每个项目的照片量是多少？`
3. 确认机器人能返回按项目拆分后的数量。
4. 确认本地模型一旦输出草稿/思维链内容，这些内容不会再出现在钉钉群回复里。
